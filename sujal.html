31. Smart Water Quality Monitoring üß™

Problem Statement

Drinking water supply to Residential Areas and Hospital requires continuous monitoring of pH, turbidity, chlorine levels, and contaminants from Industrial Area runoff.

Data Structures & C++ Code

A Circular Buffer maintains recent readings for trend analysis. Hash Map stores sensor calibration data and threshold limits.

[Water Quality Sensor Network]

#include <vector>
#include <unordered_map>
#include <deque>
#include <cmath>

const int BUFFER_SIZE = 100; // Recent readings window

struct WaterSample {
    long long timestamp;
    int sensor_id;
    double ph, turbidity_ntu, chlorine_ppm;
};

using QualityBuffer = std::deque<WaterSample>; // Circular buffer
using SensorThresholds = std::unordered_map<int, std::vector<double>>; // sensor_id -> [ph_min, ph_max, ...]

Application Logic

  • Trend Detection: Moving averages on QualityBuffer detect gradual contamination.
  • Immediate Alerts: Compare against SensorThresholds; if pH < 6.5 or turbidity > 5 NTU, trigger purification boost.
  • Predictive Flushing: If 3 consecutive samples exceed 80% of threshold, initiate localized water flushing.

Analysis & Complexity

Buffer maintenance is \(O(1)\) amortized per sample. Threshold checks are \(O(1)\). Trend analysis over window is \(O(W)\) where \(W=100\). [file:1]

32. Citizen Feedback Analytics Platform üì±

Problem Statement

City services (Market, Park, Residential) need real-time citizen feedback processing for potholes, lighting, cleanliness via mobile/web apps.

Data Structures & C++ Code

Priority Queue prioritizes urgent issues (safety > cleanliness). Spatial Index (QuadTree concept via map) for location clustering.

[Feedback Heatmap]

#include <queue>
#include <map>
#include <string>

struct Feedback {
    std::string issue_type; // "pothole", "lighting", "garbage"
    int priority; // 1=critical safety, 5=minor
    double lat, lon;
    long long timestamp;
    bool operator>(const Feedback& other) const {
        return priority > other.priority;
    }
};

using FeedbackPQ = std::priority_queue<Feedback, std::vector<Feedback>, std::greater<Feedback>>;
using GeoCluster = std::map<std::pair<int,int>, int>; // grid_x_y -> count

Application Logic

  • Triage: New feedback added to FeedbackPQ; highest priority dispatched first.
  • Clustering: GeoCluster identifies problem hotspots (3+ reports in 100m grid cell).
  • Trend Reports: Daily aggregation shows service request patterns by zone.

Analysis & Complexity

Priority queue operations \(O(\log F)\) for \(F\) feedback items. Spatial clustering \(O(F)\) with grid hashing. [file:1]
