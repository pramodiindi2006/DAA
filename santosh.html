11. Smart Retail Analytics & Demand Forecasting (Mall / Showroom / Market)

Problem Statement

Mall/showroom/market owners need demand forecasts, footfall heatmaps and product-level insight to optimize inventory, promotions and layout.

Data Structures & C++ Code

#include <vector>
#include <unordered_map>
#include <string>
#include <deque>

// Footfall counter per zone
struct ZoneCount {
    int zone_id;
    long long timestamp; // unix ts
    int count; // people entering in interval
};

// Sales record
struct Sale {
    std::string product_id;
    int zone_id;
    long long timestamp;
    int qty;
    double price;
};

// Time-windowed aggregator
using ZoneSeries = std::deque<ZoneCount>; // roll-up sliding window
using SalesMap = std::unordered_map<std::string, std::vector<Sale>>; // product -> sales

Application Logic

Ingest sensor counts into ZoneSeries, aggregate SalesMap, compute rolling averages and generate restock/promotional suggestions using weighted moving averages.

Analysis & Complexity

Sliding-window updates are O(1) amortized; computing rolling averages O(Z) for Z zones; product aggregations O(P) where P is number of SKUs within the analysis window.

12. Visitor Wayfinding & Local Discovery (Hotel / Mall / Park / Helipad)

Problem Statement

Visitors (hotel guests, tourists) need contextual navigation, POI recommendations, and multimodal directions (walk → bus → rail → helipad).

Data Structures & C++ Code

#include <map>
#include <vector>
#include <string>
#include <queue>

struct POI {
    int id;
    std::string name;
    double lat, lon;
    std::vector<std::string> tags; // museum, food, view
};

using POIIndex = std::map<std::string, std::vector<int>>; // tag -> POI ids

Application Logic

Index POIs by tags. Combine nearest-neighbor search (KD-tree recommended) plus tag filters, then compute multimodal route suggestions using short Dijkstra queries on the integrated transport graph.

Analysis & Complexity

Tag lookup O(1) (hash) then NN search O(log N) with KD-tree; route computation O(E log V).

13. Predictive Road & Pavement Maintenance (Roads / Bridges / Dam Access)

Problem Statement

Prioritize maintenance by wear, traffic and safety risk to reduce failures and cost.

Data Structures & C++ Code

#include <vector>
#include <map>

struct RoadSegment {
    int id;
    double condition_index; // 0 (bad) to 1 (excellent)
    int daily_traffic;
    bool has_bridge;
};

using SegmentMap = std::map<int, RoadSegment>;

Application Logic

Update condition_index from sensor/inspection reports. Compute priority = (1-condition)*traffic_factor*bridge_factor. Select top-K segments for maintenance scheduling.

Analysis & Complexity

Updating scores is O(S) for S segments; selecting top-K via heap is O(S log K).

14. Smart Streetlighting & Adaptive Lighting (Roads / Park / Stadium)

Problem Statement

Reduce energy and light pollution while keeping safety: dim lights in low-use hours and brighten on presence or events.

Data Structures & C++ Code

#include <unordered_map>
#include <vector>

struct LightNode {
    int id;
    double power_draw; // watts
    bool is_on;
    double brightness; // 0.0 - 1.0
    int motion_count_last_min;
};

using LightNetwork = std::unordered_map<int, LightNode>;

Application Logic

Motion sensors or event signals set `brightness`. If `motion_count_last_min` or scheduled event is active, set brightness to full; otherwise dim. Batch updates used to shave grid-level peaks.

Analysis & Complexity

Per-node update O(1); network-level adjustments O(N) but can be batched for efficiency.

15. Public Wi-Fi / Bandwidth Orchestration (Market / Railway Station / Stadium)

Problem Statement

Provide managed connectivity with fair usage, priority for emergency/official traffic, and analytics for monetization.

Data Structures & C++ Code

#include <unordered_map>
#include <queue>
#include <string>

struct UserSession {
    std::string session_id;
    int zone_id;
    long long start_ts;
    long long last_active_ts;
    int bytes_used;
    int priority; // 0 normal, 1 emergency/official
};

using SessionMap = std::unordered_map<std::string, UserSession>;

Application Logic

Track sessions, enforce QoS (elevate police/hospital traffic), implement fair-queueing or throttling, and produce usage reports for zone monetization and capacity planning.

Analysis & Complexity

Session lookup O(1); per-packet queuing complexity varies with policy structure (commonly O(log N) for priority queues).

16. Irrigation Scheduling & Precision Farming (Farm)

Problem Statement

Optimize irrigation to save water and improve yields using soil moisture, weather forecasts and dam release schedules.

Data Structures & C++ Code

#include <vector>
#include <map>

struct Plot {
    int plot_id;
    double soil_moisture; // 0-1
    double crop_coefficient; // Kc
    double last_irrigation_ts;
};

using FarmPlots = std::map<int, Plot>;

Application Logic

Compute evapotranspiration (ET) estimates and required irrigation volume per plot: need = ET * Kc - soil_available. Schedule irrigation constrained by pump capacity and dam release.

Analysis & Complexity

Per-plot calculation O(1). Scheduling for M plots with capacity constraints solved greedily in O(M log M).

17. Industrial Emissions & Compliance Monitoring (Factory / Industrial Area)

Problem Statement

Real-time compliance system: log emissions, trigger alarms when limits exceeded, and produce audit-grade records.

Data Structures & C++ Code

#include <vector>
#include <unordered_map>
#include <string>

struct EmissionSample {
    long long ts;
    std::string pollutant; // CO2, NOx, SO2
    double concentration;  // ppm or mg/m3
    int source_id;
};

using EmissionLog = std::vector<EmissionSample>;
using Thresholds = std::unordered_map<std::string, double>; // pollutant -> max allowed

Application Logic

Stream sensor readings to EmissionLog. Run windowed statistics; if thresholds crossed, trigger mitigation (throttle process), notify regulators and persist event logs for audits.

Analysis & Complexity

Windowed aggregation per pollutant O(1) per sample using running sums; alerting constant-time per event.

18. Cold-Chain & Perishable Logistics (Farm → Market → Railway)

Problem Statement

Ensure perishable produce maintains temperature during transit; match shipments to refrigerated vehicles and plan fastest routes to reduce spoilage.

Data Structures & C++ Code

#include <vector>
#include <tuple>
#include <queue>

struct Shipment {
    int id;
    int origin_plot;
    int dest_market;
    long long pick_ts;
    double max_temp; // allowable max
    double current_temp;
};

struct Vehicle {
    int id;
    double capacity;
    bool has_refrigeration;
    double current_temp;
};

using VehiclePQ = std::priority_queue<std::pair<double,int>, std::vector<std::pair<double,int>>, std::greater<>>; // (temp_match_score, vehicle_id)

Application Logic

Match shipments to refrigerated vehicles by compatibility and route via live traffic graph; if temperature drifts, trigger reroute or priority unloading.

Analysis & Complexity

Matching via PQ O(V log V) for V vehicles; routing O(E log V). Continuous telemetry checks O(1) per report.

19. Event & Venue Management (Stadium / Park / Showroom)

Problem Statement

Events require crowd control, dynamically adjusted transport, temporary parking and emergency staffing.

Data Structures & C++ Code

#include <vector>
#include <map>

struct Event {
    int id;
    long long start_ts;
    long long end_ts;
    int expected_attendance;
    std::vector<int> related_zones; // parking, stations
};

using EventCalendar = std::map<long long, Event>; // start_ts -> event

Application Logic

When event scheduled, provision extra public transport runs, reserve parking slots, pre-deploy medics/security, and adjust traffic signals to manage arrival/departure curves.

Analysis & Complexity

Event provisioning O(Z) by related zones; IPTS insertion cost depends on time-expanded graph complexity (O(V_TE log V_TE)).

20. Smart Surveillance & Anomaly Detection (Bank / Railway Station / Mall)

Problem Statement

Automated CCTV analytics for crowd anomalies, unattended objects and suspicious behavior in high-value zones.

Data Structures & C++ Code

#include <unordered_map>
#include <vector>
#include <string>

struct CameraFrameMeta {
    long long ts;
    int camera_id;
    std::vector<std::pair<int,int>> bounding_boxes; // objects
    std::vector<std::string> detected_labels; // person, bag, vehicle
};

using CameraFeedIndex = std::unordered_map<int, std::vector<CameraFrameMeta>>; // cam -> frames

Application Logic

Run on-edge detectors or central models. Maintain short-term object tracks and compute anomalies (loitering, crowd spikes, left object). On anomaly, escalate to human operator with frames & location.

Analysis & Complexity

Per-frame detection cost depends on chosen model. Tracking and rule evaluation per object approx O(k) where k is number of tracked objects. Alerting O(1).
